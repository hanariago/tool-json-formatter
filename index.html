JSON 포매터 사용법

    
개발 중 API 응답이나 설정 파일의 JSON이 한 줄로 뭉쳐 있어 읽기 어려울 때, 이 도구에 붙여넣으면 들여쓰기로 정렬되고 유효성까지 즉시 확인됩니다. 따옴표 누락이나 쉼표 오류가 있으면 몇 번째 줄에서 문제가 생겼는지 알려줍니다.

    
붙여넣기 외에도 폴더나 여러 파일을 한 번에 열 수 있습니다. 프로젝트 폴더를 선택하면 그 안의 JSON과 Markdown 파일이 트리로 보이고, 클릭하면 왼쪽에 원본, 오른쪽에 보기 좋게 정리된 결과(JSON은 정렬·검증, Markdown은 렌더링)가 나란히 표시됩니다. 모든 처리는 브라우저 안에서만 이루어져 어떤 데이터도 서버로 전송되지 않습니다.

  

  

    
자주 묻는 질문

    
내 JSON 데이터나 파일이 서버로 전송되나요?
아니요. 모든 처리는 브라우저 안에서만 이루어지며, 입력한 JSON이나 선택한 파일은 어떤 서버로도 전송되지 않습니다.

    
어떤 파일을 열 수 있나요?
폴더 또는 여러 파일을 선택할 수 있고, 그 안의 .json 과 .md/.markdown 파일이 목록에 표시됩니다. JSON은 포맷·검증해 보여주고, Markdown은 미리보기로 렌더링합니다.

    
들여쓰기 2칸과 4칸은 어떻게 다른가요?
정렬된 JSON의 들여쓰기 폭만 달라집니다. 2칸은 더 촘촘하게, 4칸은 더 넓게 보입니다. 데이터 내용은 동일합니다.

    
최소화(Minify)는 무엇인가요?
줄바꿈과 공백을 모두 제거해 JSON을 한 줄로 압축합니다. 전송 용량을 줄이거나 코드에 한 줄로 붙여넣을 때 유용합니다.

    
모바일에서도 쓸 수 있나요?
네. 모바일에서는 좌우 분할이 상하 분할로 바뀌고, 파일 목록은 접고 펼 수 있는 드로어로 표시됩니다.
